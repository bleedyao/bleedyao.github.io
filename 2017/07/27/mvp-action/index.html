mvp 基本信息

  • MVP模式（Model-View-Presenter）可以说是MVC模式（Model-View-Controller）在Android开发上的一种变种和进化模式。后者大家可能比较熟悉，就算不熟悉也可能或多或少地用到过MVC。

mvp 核心思想

  • 在 mvp 中，由于业务逻辑堆砌在控制层中，因此代码会显得很臃肿，mvp 的使用可以有效的将控制层削减
  • mvp 的核心思想是把 Activity 中的 UI 逻辑抽象成 View 接口，把业务逻辑抽象成 Presenter 接口，Model 类还是原来的 Model。
    • 这样的话，Activity的工作的简单了，只用来响应生命周期，其他工作都丢到Presenter中去完成。
